Visual Style

Typography

Because so much of Wikipedia and wiki projects rely on writing and reading, typography is a key component of Wikimedia design. You need to consider the typeface, size, style and spacing of your text to make it achieve a good readability.

Readable content

We want content to be readable for everyone, in all conditions. Our typographic choices make our content accessible, presented in a neutral way and convey its reliability.

Layout text

Color contrast. When using text, make sure that it provides enough contrast to be read comfortably. Check the contrast between the colors used for text and its background. Make sure to provide at least level AA contrast. The color palette provides the contrast levels for pure white and black surfaces, but you need to check the contrast on other combinations.

Text spacing. How text is placed in space affects its readability. Follow these considerations for text paragraphs:

  • Line length should be no longer than 75 characters.
  • Line height should be 1.6 times the size of the font used.

Making text fit. Content will be available in multiple languages, so text length will vary for a given piece of information across languages. Strategies to consider:

  • Don’t crowd the UI. Be short and clear when naming interface actions.
  • Dynamic layout. Make containers expandable so that they can fit the content.
  • Dynamic text. Adjust the size depending on the content. Use a smaller font-size for long content.
  • Clipping. Clipping the text with ellipsis (…). Only if there is no risk of missing important information or the complete information is reachable through a clear alternative means.
